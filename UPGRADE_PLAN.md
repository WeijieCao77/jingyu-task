## 一、操作层面（对话体验：不同情境不同回答）

**现状（本轮已落地，来自真机电话反馈 FR-3~9）**
- 温和开场白（不再冷冰冰三连问）；字母带消歧词回读（B,Boy 的 B）；只复述被更正的那一项；
- 手机号 11 位校验、位数不对请重报；单位不在名单 → 先确认再转人工；无具体公司不强逼/不编造；
- 转人工同时推送门卫手机；登记完静默自动挂断 + 单通最长时长兜底。

**计划（分阶段）**
1. **情境化话术库**：按事由（送货/拜访/面试/参观/快递/维修/接送…）走不同的追问与确认重点
   （如"送货"问卸货点、"面试"问联系人、"参观"可无具体公司）。做法：prompt 里加事由分支 + 可选每租户自定义话术。
2. **异常/情绪处理**：访客不耐烦、方言重、环境嘈杂、反复听不清 → 更早转人工或换确认策略。
3. **多语言**：检测到英文/其他语言访客自动切换应答语言（realtime 原生支持多语，pipeline 需切 STT/TTS）。
4. **更强号码/车牌纠错**：已做字母消歧 + 手机校验；可加车牌受限词表、ASR 后处理、热词。
5. **挂断/计时打磨**：看门狗参数按真机调；超时前一句礼貌提示。

## 二、模型架构层面（效率 / 准确 / 省 token 与成本）

**现状（本轮已落地）**
- realtime 默认（首句 ≈1.4s）+ pipeline 一行回退；模型全 env 可换（`LLM_BASE_URL`）；
- **数据查询强制先调工具**（杜绝"凭空答 0/拒答"）+ **模型分层** `GUARD_QUERY_MODEL`（查询可用与大脑不同档位）。

**计划（分阶段）**
1. **模型分层全面化**：大脑 / 数据查询 / 纠错各挑合适档位——简单问答用便宜模型，复杂推理用强模型。
   已开 `GUARD_QUERY_MODEL`；可扩展 STT/TTS、"纠错/复述"专用小模型。
2. **省 token / 省成本**：
   - prompt 精简 + **Anthropic 提示缓存**（system 长文走 `cache_control`，多轮省钱）；
   - 工具结果截断（list 限行数，已限 20）、`max_tokens` 上限；
   - **按需才上 realtime**：短问答/低峰可用 pipeline，省 5~10 倍/分钟成本（已可一行切）；
   - 来电号码预填手机（省一轮对话 = 省 token + 缩短通话）。
3. **准确性**：确定性层兜底（名单/省份/手机校验已做）+ 关键字段二次确认 + 可选受限词表 ASR。
4. **成本可观测**：每通话记录 token / 时长 / 估算成本，落 `call_events`，Dashboard 出量化看板。
5. **回归评测集**：构建"对话质量 + 延迟 + 成本"评测脚本，换模型/改 prompt 前后自动对比（防回退）。

## 三、产品化（多租户 SaaS：客户注册手机号语音 Agent）

> 目标：不再是"一个号码一套系统"。客户**注册账号 → 拿到自己的号码 + 语音门卫 → 自助配置名单/通知/门卫 → 直接用**。

**现状**：单租户，配置走 `.env`（一套 roster/access/notify/门卫）。

**计划（分阶段，逐步实现）**
1. **阶段 1 · 多租户配置（本轮起步）**：按**被叫号码**（`sip.trunkPhoneNumber`）解析租户，
   各租户独立 `roster / 黑白名单 / 通知渠道 / 门卫口令 / 门卫手机 / 话术`。→ `tenant.py` 已落地解析基础 + schema + 测试。
2. **阶段 2 · 数据隔离**：`visits`/`call_events` 加 `tenant_id`；Dashboard/查询/导出全部按租户过滤；
   一个号码的数据另一个租户看不到。
3. **阶段 3 · 自助开通**：客户注册 → 后端用 **Twilio API 自动购号** + **LiveKit SIP 自动建 trunk/dispatch** + 绑定到该租户；
   `scripts/setup_sip.sh` 升级为按租户编排的服务。
4. **阶段 4 · 租户控制台**：网页自助配置（上传公司名单、设黑白名单、配 Telegram/企微、设门卫口令/手机、改话术），
   看本租户用量。
5. **阶段 5 · 计费与配额**：按分钟/条数计费 + 套餐 + 限额；接支付。
6. **部署**：容器化多租户 + Neon（行级 `tenant_id` 或每租户 schema）；无状态 worker 横向扩展。

**本轮已落地（阶段 1 起步）**
- `tenant.py`：`load_tenants(path)` + `resolve_tenant(called_number)`，按被叫号码取该租户的
  `roster_path / access_list_path / notify / guard_access_key / guard_phones / public_base_url` 覆盖；
  **默认关闭**（无 `TENANTS_PATH` = 现有单租户行为不变），含 `tenants.example.json` + 单测。
- 这是把"一套系统"演进成"多租户平台"的第一块地基——**不破坏现有单租户 demo**。
