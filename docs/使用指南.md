# 个人主页 + GitHub 装饰 · 操作指南

这份指南教你完成两件事：

1. **个人主页**：一个人人都能打开的网页，含自我介绍、职业照、简历下载（用 GitHub Pages 免费托管）。
2. **装饰 GitHub**：让你的 GitHub 个人页（`github.com/weijiecao77`）顶部出现一张漂亮的介绍卡片（Profile README）。

> 这些文件都在仓库的 `docs/` 文件夹里。本次改动在分支 `claude/inspiring-euler-4zmiqt` 上，并已开了一个 Pull Request。**先把这个 PR 合并到 `main` 分支**，再做下面的步骤。

---

## 一、上线你的个人主页（GitHub Pages）

### 步骤 1：开启 GitHub Pages
1. 打开仓库 `weijiecao77/jingyu-task` → 顶部 **Settings**（设置）。
2. 左侧菜单点 **Pages**。
3. **Build and deployment** → **Source** 选 **Deploy from a branch**。
4. **Branch** 选 `main`，文件夹选 **`/docs`**，点 **Save**。
5. 等 1～2 分钟，页面顶部会出现网址，通常是：

   **https://weijiecao77.github.io/jingyu-task/**

这个网址是公开的，任何人（不用登录 GitHub）都能打开。✅

### 步骤 2：放上你的职业照和简历
进入 `docs/assets/` 文件夹（网页或本地都行），放入两个文件，**文件名必须完全一致**：

| 文件名 | 用途 |
|---|---|
| `profile.jpg` | 你的职业照（建议正方形，如 600×600） |
| `resume.pdf`  | 你的简历（覆盖现有占位文件） |

最简单的做法：在网页上进入 `docs/assets/` → **Add file → Upload files** → 拖入文件 → **Commit changes**。

> 没放照片也没关系，主页会先显示一个默认占位头像。

### 步骤 3：填上你的名字和自我介绍
打开 `docs/index.html`，找到顶部带 ★ 标记的 **`CONFIG`** 部分，只改这一段就够了：

```js
const CONFIG = {
  name:    { zh: "你的名字",        en: "Your Name" },
  title:   { zh: "你的职位 / 头衔",  en: "Your Title / Role" },
  tagline: { zh: "一句话介绍",       en: "One-line tagline" },
  about:   { zh: "中文自我介绍…",    en: "English about…" },
  ...
  contacts: [
    { type: "github", label: "GitHub", url: "https://github.com/weijiecao77" },
    { type: "email",  label: "Email",  url: "mailto:你的邮箱@example.com" }
  ]
};
```

- `zh` 是中文、`en` 是英文（主页右上角可一键切换中/英）。
- 不想公开邮箱：把 `email` 那一行删掉即可。
- 改完 **Commit**，等一两分钟主页就会更新。

## 二、装饰你的 GitHub 主页（Profile README）

这一步会让别人访问 `github.com/weijiecao77` 时，最上面出现你的介绍卡片。

1. 在 GitHub 点右上角 **+ → New repository**（新建仓库）。
2. **Repository name 必须等于你的用户名：`weijiecao77`**（一字不差，GitHub 才会识别为特殊仓库）。
3. 勾选 **Public**，并勾选 **Add a README file**，点 **Create repository**。
4. 打开模板文件 `docs/profile-readme.md`，把里面**横线以下、注释标记之间**的内容复制到新仓库的 `README.md`。
5. 把里面的占位文字、链接换成你自己的，**Commit**。
6. 打开 `github.com/weijiecao77` 即可看到效果。

模板已经内置了：技能徽章、GitHub 数据统计图、访客计数，以及指向你个人主页的链接。

> 想要更多徽章/装饰：
> - 徽章生成：https://shields.io
> - 现成徽章集合：https://github.com/Ileriayo/markdown-badges
> - 更多花样：搜索 “awesome-github-profile-readme”

## 常见问题

**Q：网页打不开 / 404？**
A：确认 Settings → Pages 的 Source 是 `main` 分支 + `/docs` 文件夹；刚保存需等 1～2 分钟。

**Q：照片不显示？**
A：文件名要正好是 `profile.jpg`，并放在 `docs/assets/` 下。若是 PNG，改 `index.html` 里的 `CONFIG.photo` 为 `assets/profile.png`。

**Q：能用自己的域名吗？**
A：可以。在 Settings → Pages → Custom domain 填你的域名，并在域名服务商加 CNAME 记录。

**Q：个人主页和 Profile README 有什么区别？**
A：Profile README 是 GitHub 站内顶部的卡片；个人主页是一个独立网页（有自己的网址，可放简历下载、照片等更丰富的内容）。两者互不冲突，建议都做，并互相加链接。
