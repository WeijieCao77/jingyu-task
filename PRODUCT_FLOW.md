## 一、一句话定位

园区**未登记车辆**呼入（电话/扫码/浏览器）→ AI 门卫用**自然中文对话**采集 4 项信息（车牌·来访单位·手机号·来访事由）→ 推送保安 → 保安确认 → 抬杆。**从 AI 开口到推送 ≤25 秒**，每次通行留痕可分析。

---

## 二、三种接入方式（之后对话与后端完全一致）

| 方式 | 访客做什么 | 状态 | 适合 |
|---|---|---|---|
| 📞 电话（任务原意） | 看入口号码 → 拨打 | 代码就绪，需 Twilio/SIP 配置 | 零学习成本人群 |
| 📷 扫码 | 扫入口二维码 → 手机网页对话 | 已做好 | 无需电话基础设施 |
| 💻 浏览器麦克风 | 打开网页点接入 | 已做好（最省事验收） | 本地演示 |

三者最终汇入**同一个 LiveKit room + 同一个 Agent**——换接入方式不改对话逻辑。

## 三、访客（送货人员）端：会经历/看到什么

### 📞 电话：纯语音，零界面
就是一通普通电话：
```
AI：您好，请问车牌号多少，今天找哪家公司，什么事儿？
访客：沪A12345，来蓝色鲸鱼送货的
AI：好嘞，手机号方便留一下吗？
访客：138……
AI：好的！沪A12345，蓝色鲸鱼送货，已通知门卫，请稍等放行。
```
**像跟真人保安通话，没有任何屏幕。** 优点零学习成本；缺点无凭证、看不到结果。

### 📷/💻 扫码或浏览器：极简一页
访客看到的实际页面：
```
        🐳 和 AI 门卫对话
  点按钮、允许麦克风，就能直接说话登记——无需电话
            [ 📞 接入门卫 ]
```
点击后：
```
   ● (跳动的绿色麦克风圈)
   ✅ 已接入，门卫会先开口——请直接说话
```
**故意做得极简**：访客不看字段、不打字，只管说。看不到采集了什么，也看不到后端。

> 对话体验硬标准：一句话能给多项就不重复问、能被打断、像真人门卫而非表单（详见 DESIGN.md 对话体验标准）。

## 四、一通电话内部发生了什么（25 秒时间线）

```
接通 → AI 开口(计时开始)
  → 访客说话 → STT 转文字 → LLM 边听边用工具填槽(车牌/单位/手机/事由)
  → 缺哪项补问一句（可被打断）→ 四项齐 → 复述确认
  → 后端：写入数据库 + 推送保安(带"确认放行")   ← 这一刻 ≤25 秒
  → 保安确认 → 校验 token → 抬杆(海康 ISAPI / demo stub)
```
组件：LiveKit（电话/打断/并发）· STT（默认 OpenAI zh）· LLM（默认 GPT-4o-mini，可切 Claude）· TTS（默认 OpenAI，可切 Azure zh-CN）。

## 五点五、转人工 / 保安介入（v1 已做）

因为三种接入（电话/扫码/浏览器）**都进同一个 LiveKit 房间**，转人工 = 让真人也进这个房间：

- **触发方式**：① 访客说"找真人/转人工" ② AI 连续听不懂/情况异常 → AI 调 `request_human` → Dashboard 实时流出现 **⚠️ 转人工** 事件，旁边一个 **📞 介入通话** 按钮。
- **保安介入**：点"📞 介入通话" → 打开 `/guard_call` 页面 → 接入访客所在房间 → **AI 自动让位**（说一句"门卫师傅来了"然后退出房间）→ 保安与访客**实时对讲**。
- **电话场景**：同理可用（LiveKit 还支持 SIP 把通话转接到保安手机）。
- **最简兜底**：扫码场景我们已采集手机号，保安也可直接回拨。

> 状态：触发/事件/介入页/AI 让位逻辑已实现并单测；"保安加入→AI 退出"的实时表现需本地一次真跑确认（见 SMOKE_CHECK.md）。

## 六、采集了什么、能驱动什么

- 每次登记 = 一条 `visits` 记录；**每次开闸都有时间戳**（`call_events` + `confirmed_at`）。
- **回访识别**：车牌 / 手机号 / 姓名 都能认（手机=人、车牌=车），老客户开场直接确认不重问。
- 驱动：常客画像、自然语言查询、回访——后续优化的数据地基已就位。

## 八、产品决策点（PM 视角待拍板）

都不是技术问题，是"产品该怎么设计"：

1. **放行：人工点 vs 自动放行？** 现为人工（安全可控）。是否"可信访客自动放行、仅异常人工"？
2. **要不要给访客回执？** 现仅口头"已放行"，无短信/凭证。
3. **"来访单位"是否也通知被访公司**（而非只通知保安）？真实园区里蓝色鲸鱼可能也想知道。
4. **异常兜底**：转人工 v1 已做（访客要求或 AI 卡住→保安一键介入同一通话，AI 让位，见 §五点五）。待你定的是触发策略松紧、以及要不要加黑/白名单。
5. **姓名必采否？** 现为可选（提到才记）。
6. **隐私合规**：手机号/车牌是个人信息，是否脱敏、设留存期、限制谁能查？
7. **多入口/高峰**：一个号 vs 每入口一个号；多车同时呼入体验。
8. **回访认错人**：公司车换司机已设计成"确认而非假设"；要不要更保守（只认手机不认车牌）？

> 反馈方式：挑任意一条给意见即可，或提全新的点，我据此调整并记入 CHANGELOG。
